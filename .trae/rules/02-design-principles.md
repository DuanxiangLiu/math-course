# 设计原则

## 1. 教学设计原则

### 1.1 建构主义学习（做中学）
- 每个重要概念都应该考虑是否可以添加互动演示
- 让学生通过操作、探索、发现来构建数学理解
- 避免直接给出结论，而是引导学生通过操作自己发现规律
- 例如：有理数加法，让学生拖动数轴上的点，观察结果

### 1.2 螺旋式上升
- 在不同章节间建立联系，提供"知识回顾"链接
- 在练习中设计综合题，融合多个章节的知识点
- 同一概念在不同阶段反复出现，逐步深化

### 1.3 完整学习闭环
- 每个章节开始时明确学习目标
- 概念讲解后紧跟例题和练习
- 练习提供即时反馈和详细解析
- 记录错题，提供重练机会
- 章节结束时提供总结和检测

## 2. 交互设计原则

### 2.1 即时反馈
- 交互演示实时更新结果
- 练习题点击后立即显示对错
- 错误时给出详细解析，不仅是"错了"
- 正确时给予肯定和鼓励

### 2.2 进度保存
- 使用 localStorage 保存章节完成状态
- 记录错题，支持错题重练
- 在首页显示学习进度
- 支持断点续学

### 2.3 可视化设计
- 根据不同数学概念选择合适的可视化方式
- 二维几何和函数图像：可以使用 JSXGraph 或原生 Canvas
- 数学公式：考虑使用 MathJax 或 KaTeX
- 保持灵活性，按需引入，避免过度工程化

## 3. 技术实现原则

### 3.1 轻量级纯前端架构
- 不依赖后端服务器，纯 HTML/CSS/JavaScript 实现
- 易于部署，GitHub Pages 即可托管
- 维护成本低，不需要服务器运维
- 可离线使用，适合各种环境
- 使用 localStorage 保存学习进度和用户数据

### 3.2 多种可视化工具整合
- 根据不同数学概念选择合适的可视化工具
- 二维几何和函数图像：可以使用 JSXGraph 或原生 Canvas
- 三维几何：可以使用 Three.js（如需要）
- 数学公式：使用 MathJax 或 KaTeX
- 保持灵活性，按需引入，避免过度工程化
- 优先使用轻量级方案，避免引入不必要的依赖

### 3.3 响应式和无障碍设计
- 适配各种设备，支持特殊需求学习者
- 保持响应式设计，支持移动端和桌面端
- **移动端优先设计**：
  - 屏幕宽度 &lt;= 640px（手机）：优化小屏体验
  - 屏幕宽度 641px - 968px（平板）：中等屏幕适配
  - 屏幕宽度 &gt; 968px（桌面）：完整功能展示
- **侧边栏导航响应式方案**：
  - 桌面端（&gt;968px）：固定侧边栏，始终显示目录
  - 移动端（&lt;=968px）：将侧边栏改为可折叠的下拉菜单
  - 添加菜单切换按钮（.sidebar-toggle-btn），默认隐藏侧边栏
  - 点击导航链接后自动收起侧边栏
- 确保字体足够大（建议正文至少 16px），对比度足够高
- 添加 ARIA 标签，支持屏幕阅读器
- 支持键盘导航，确保所有交互元素可以通过键盘访问
- 遵循 WCAG 2.1 AA 级标准（如适用）
